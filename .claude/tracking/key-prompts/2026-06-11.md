# Key Prompts — 2026-06-11

## Reachability: worklist traversal vs Warshall closure (architecture)

**Category**: architecture
**Prompt**: "let's make a documented note about this and why we chose BFS
here. We could explore using julia to have a more optimal / fast compute of
the Warshall but unless i'm missing something, we don't need it, right?"

**Outcome**: Decided against the Julia/Warshall exploration; wrote
`docs/reachability.md` documenting the rationale and linked it from the
call-graph comment block in `yggdrasil.shen`.

**Decided against and why**: A fast (Julia/bitset) Warshall closure was
considered and rejected — not because closure is too slow to implement
well, but because it answers the wrong question. The shaker needs one
multi-seed reachability query per shake (O(V+E) ≈ 6.7K ops over V=1,129,
E=5,619), while any closure is Ω(V²) just to emit output the pipeline
never reads. An external Julia step would also break stage 1's "runs on
any certified Shen" contract for zero measured benefit — profiling showed
the cost lives in graph *construction* (solved by the disk cache) and KL
I/O, not traversal. Revisit only for dense many-pair queries, and even
then prefer Tarjan SCC/condensation over Warshall.
